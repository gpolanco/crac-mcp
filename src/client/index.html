CRAC MCP Server

Model Context Protocol Server v1.0.0

Configuración

Cursor

Agrega la siguiente configuración a tu archivo de configuración de MCP en Cursor.

~/.cursor/mcp.json 
{
  "mcpServers": {
    "crac": {
      "url": "https://crac-mcp-production.up.railway.app/mcp",
      "type": "http"
    }
  }
}

Ubicación del archivo: ~/.cursor/mcp.json o en la configuración de Cursor Settings.

Tools

hello

Say hello to someone

get_info

Retorna información genérica del servidor MCP

get_crac_rules ⭐ Auto-detect

Obtiene las reglas y convenciones del monorepo CRAC basándose en el contexto proporcionado. El tool detecta automáticamente qué reglas son relevantes (testing, estructura, endpoints, estilo de código).

Detección automática:
• "implementa los test de X" → Reglas de testing
• "crea la estructura de Y" → Reglas de estructura
• "implementa el servicio Z" → Reglas de endpoints
• "crea un componente" → Reglas de code style

Ejemplos de uso:
get_crac_rules con: {"context": "implementa los test de EmailService"}
get_crac_rules con: {"context": "crea la estructura para un nuevo módulo roles"}
get_crac_rules con: {"context": "implementa el servicio de usuarios"}

Nota: El agente debe usar este tool automáticamente cuando detecte que necesita entender las convenciones CRAC. El tool está diseñado para ser llamado proactivamente por el agente.

Prompts

greet

Say hello to someone

Ejemplo:
crac/greet con argumento: {"name": "World"}

dev_task RAG

Generate context-aware development prompts using RAG. Parses natural language commands and searches for relevant context from the monorepo.

Ejemplos de uso:
crac/dev_task con: "dev rac implementa la nueva sección booking-search"
crac/dev_task con: "test partners add unit tests for auth flow"
crac/dev_task con: "refactor global improve code structure"

Formatos soportados:
• Tools: dev, test, refactor, fix, update
• Scopes: rac, partners, global, web, mobile, suppliers, notifications, queues

generate_tasks RAG + Templates

Generate structured task documentation (PRD → Tasks → Subtasks) using templates and RAG context. Searches for templates and monorepo context to create a complete markdown document.

Ejemplos de uso:
crac/generate_tasks con: "generate-tasks partners nueva sección settings"
crac/generate_tasks con: "gen-tasks rac implementar buscador de reservas"

Este prompt genera un documento completo con PRD, Tasks y Subtasks listo para revisar y copiar.
